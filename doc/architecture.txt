================================
  PROJECT OVERVIEW
================================

  Dual-market stock scanner with:
  - US pipeline driven by a prebuilt CSV universe
  - India pipeline driven by live NSE symbol discovery
  - Threaded chunk scanning in both scanners
  - Incremental daily re-scanning based on prior score/failure history
  - Daily result snapshots for dashboard comparisons
  - Browser dashboard with watchlist, sparkline, delta, copy, and CSV export

================================
  SCANNER INTERNALS
================================

  US scanner:
  - reads static descriptive fields from CSV
  - fetches 2 years of OHLCV via yfinance
  - applies 5 primary gates
  - computes 0-7 score
  - stores closes_30d for dashboard sparkline rendering

  India scanner:
  - loads symbols from NSE
  - fetches OHLCV from Fyers or yfinance fallback
  - applies 5 primary gates
  - computes 0-9 score
  - adds delivery %, circuit information, cap category
  - stores closes_30d for dashboard sparkline rendering

  Both scanners:
  - use ThreadPoolExecutor(max_workers=4) per chunk
  - keep results/scanned state behind threading.Lock()
  - preserve per-ticker retry handling inside worker threads
  - print ticker output as each worker completes
  - persist latest state + full run history in SQLite
  - archive every scan event in a separate permanent SQLite DB


================================
  INCREMENTAL RE-SCAN RULES
================================

  Re-scan when ANY are true:
  - never scanned before
  - previous score >= 3
  - exactly one primary gate failed last time
  - last scan is older than 3 days

  Skip only when BOTH are true:
  - 2 or more primary gates failed last time
  - last scan is within 3 days


================================
  DASHBOARD ARCHITECTURE
================================

  dashboard.html
  │
  ├── fetch current market file:
  │     scanner_results.json or india_results.json
  │
  ├── fetch newest older snapshot available:
  │     scanner_results_YYYY-MM-DD.json
  │     india_results_YYYY-MM-DD.json
  │
  ├── backend interaction (api.py):
  │     - sends command launch requests
  │     - receives live stdout stream via EventSource
  │     - supports US and India specific tool presets
  │
  ├── localStorage:
  │     key = signalscan_watchlist
  │     values = ["US:AAPL", "IN:RELIANCE", ...]
  │
  ├── table features:
  │     - star toggle per row
  │     - watchlist-only filter with count
  │     - score badge + score delta
  │     - sparkline from closes_30d
  │     - copy visible tickers
  │     - export visible rows to CSV
  │
  └── detail panel:
        - score summary
        - condition status
        - metrics grids
        - external links


================================
  RESILIENCE FEATURES
================================

  - Atomic JSON save after each chunk
  - Snapshot copy after each full scan
  - Snapshot pruning to latest 7 files
  - Rate-limit retries per worker
  - Chunk pauses preserved
  - Dashboard retry when live JSON is mid-write
  - Watchlist persists in browser across refresh/restart
