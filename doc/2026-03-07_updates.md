# SignalScan Updates — 2026-03-07

## Scanner Features Added

### Parallel chunk scanning

- `scanner.py` and `india_scanner.py` now process each chunk with `ThreadPoolExecutor(max_workers=4)`
- Shared `results` and `scanned` state is protected with `threading.Lock()`
- Per-ticker retry handling remains inside each worker thread
- Existing chunk pauses and atomic per-chunk JSON save behavior were preserved

### Incremental daily re-scanning

- `scanner_scanned.txt` and `india_scanned.txt` moved from plain ticker lines to structured history entries:

```text
TICKER,score=5,failed=,date=2026-03-07
TICKER,score=0,failed=P1|P5,date=2026-03-06
```

- Re-scan rules added:
  - re-scan if score was `>= 3`
  - re-scan if only one primary condition failed
  - re-scan if last scan is older than 3 days
  - re-scan if never scanned before
- Skip only when:
  - 2 or more primary conditions failed
  - and the last scan was within 3 days

### 30-day close history for charts

- `scanner.py` and `india_scanner.py` now include:

```python
'closes_30d': [round(float(x), 2) for x in close.tail(30).tolist()]
```

- This powers the dashboard sparkline/chart column

### Daily result snapshots

- After a full scan completes, the scanners now also save:
  - `scanner_results_YYYY-MM-DD.json`
  - `india_results_YYYY-MM-DD.json`
- Snapshot retention is automatic:
  - only the latest 7 daily snapshots are kept

## Dashboard Features Added

### Persistent watchlist

- Added per-row star toggle before the ticker
- Watchlist entries are stored in `localStorage` under `signalscan_watchlist`
- Market-aware keys are used:
  - `US:TICKER`
  - `IN:TICKER`
- Added `★ Watchlist (N)` filter button with live count

### Inline chart column

- Added a `Chart` column between `Score` and `Signals`
- Renders an inline SVG sparkline at `80x24`
- Line color:
  - green `#00e5a0` when last close >= first close
  - red `#ff4d4d` when last close < first close
- Last point includes a filled dot
- Falls back to `—` when `closes_30d` is missing

### Score delta badges

- Dashboard now compares current scores to the most recent available older snapshot
- Added score-delta badges next to the score:
  - `↑+N`
  - `↓-N`
  - `NEW`
- If no prior snapshot is available, no badge is shown

### CSV export

- Added `↓ Export CSV` button next to `Copy Tickers`
- Exports only the currently visible / filtered rows
- Output filename format:
  - `signalscan_US_YYYY-MM-DD.csv`
  - `signalscan_IN_YYYY-MM-DD.csv`

### TradingView hover popup

- Added TradingView hover preview on the ticker cell
- Popup behavior:
  - delayed open on hover
  - floating reusable popup
  - close on popup leave or close button
- India symbols use:
  - `NSE:TICKER`
- US symbols now use exchange-prefixed TradingView symbols when available:
  - `NASDAQ:TICKER`
  - `NYSE:TICKER`
  - `AMEX:TICKER`
- US exchange mapping is read client-side from:
  - `us_stocks_100m_10b_full.csv`

## Follow-Up Fixes Added Today

### SQLite scan state

- `scanner.py` and `india_scanner.py` now persist scan state in `scan_state.db`
- Added SQLite tables for:
  - `latest_scan_state`
  - `scan_history`
  - `scan_runs`
  - `scan_snapshots`
- Legacy `scanner_scanned.txt` and `india_scanned.txt` logs are imported into SQLite on first run and no longer act as the primary state store

### Snapshot 404 reduction

- Removed blind historical snapshot probing in the dashboard
- Snapshot comparison now only uses actual discovered snapshot files
- This prevents repeated 404 requests for guessed snapshot dates

### No-cache dashboard serving

- `start_dashboard.sh` now serves dashboard assets with no-cache headers
- `dashboard.html` also includes no-cache meta tags
- This reduces stale-browser JS issues after dashboard changes

### Favicon support

- Added `favicon.svg`
- Linked favicon from `dashboard.html`
- Prevents repeated browser requests for missing `/favicon.ico`
