# Contributing to US&IN Scanner

First off, thank you for considering contributing to the US&IN Scanner! 

## How Can I Contribute?

### Reporting Bugs
If you find a bug:
- Ensure the bug was not already reported by searching through issues.
- If you're unable to find an open issue addressing the problem, open a new one. Be sure to include a title and clear description, as much relevant information as possible, and a code sample or an executable test case demonstrating the expected behavior that is not occurring.

### Suggesting Enhancements
- Open a new issue with a clear title and description explaining the feature and why it would be beneficial.
- If you have an idea for implementation, include it in your issue description!

### Pull Requests
- Fork the repository and create your branch from `main`.
- If you've added code that should be tested, ideally ensure the tests pass.
- If you've changed APIs, update the documentation.
- Ensure any private configurations (like API keys or specific scan conditions) are **NOT** included in your commits.
- Make sure your `.scanner_secrets/` folder and `.env` files are fully excluded!

## Local Development Setup
1. Clone your fork of the repository locally.
2. Initialize your private secrets and API configs according to the `README.md`.
3. Test your changes using either scanning script before opening a Pull Request.

Thank you for contributing!
